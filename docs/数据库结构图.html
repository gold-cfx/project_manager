👥 用户表 (users)
users
存储系统用户信息，支持权限管理和用户认证
id: INT AUTO_INCREMENT PK
username: VARCHAR(50) UNIQUE NOT NULL
password: VARCHAR(64) NOT NULL
real_name: VARCHAR(50) NOT NULL
role: VARCHAR(20) DEFAULT 'user'
status: VARCHAR(20) DEFAULT 'active'
email: VARCHAR(100)
phone: VARCHAR(20)
last_login: TIMESTAMP
create_time: TIMESTAMP DEFAULT CURRENT_TIMESTAMP
update_time: TIMESTAMP ON UPDATE

🎯 数据库设计特点

设计亮点：

  • 规范化设计：遵循第三范式，减少数据冗余
  • 外键约束：确保数据完整性，支持级联删除
  • UTF-8编码：支持多语言字符存储
  • InnoDB引擎：支持事务和外键约束
  • 时间戳字段：自动记录创建和更新时间
  • 唯一索引：防止重复数据，如用户名、项目名
  • 软删除支持：通过status字段实现逻辑删除

性能优化：

  • 复合索引：dict_type + dict_key 联合索引
  • 外键索引：自动为外键字段创建索引
  • 时间索引：create_time、update_time索引
  • 状态索引：status字段索引优化查询
📋 数据字典示例

字典类型说明：

project_source: 项目来源（国家级、省部级、校级等）
project_type: 项目类型（基础研究、应用研究、开发研究等）
level: 项目级别（重大项目、重点项目、一般项目等）
status: 项目状态（进行中、已完成、已终止等）
reminder_type: 提醒类型（到期提醒、进度提醒等）
reminder_way: 提醒方式（弹窗、邮件、短信等）
